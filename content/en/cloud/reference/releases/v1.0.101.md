## Hotfix - custom-domain orgs with anchored-fallback credentials hit 1722 at callback

Custom-domain orgs without their own BYOC IdP credentials (e.g.
`exoscale.layer5.io`) ran into `meshery-cloud-1722 \state validation failed\`
immediately after GitHub redirected back to `/oidc/callback/github`. The W2
state binding stamped the request host (`exoscale.layer5.io`), but the callback
legitimately lands at the canonical host (`cloud.layer5.io`) because the
fallback chain anchors `redirect_uri` there. The validator's strict host-binding
equality refused the cross-host bounce.

### What changed

`OIDCStart` now stamps the **effective callback host** into the state payload,
mirroring `composeCallbackURL`'s precedence:

- BYOC (`cred.CallbackHost` empty) -> stamp `req.Host`
- Anchored fallback (`cred.CallbackHost` set) -> stamp `cred.CallbackHost`

So the state's host-binding holds whether the callback lands on the same custom
domain or on the canonical Provider Org host.
