## Hotfix - production sign-in restored on cloud.layer5.io (v1.0.97 regression)

Fixes a regression introduced in v1.0.97 where sign-in via GitHub / Google on `cloud.layer5.io` failed with `meshery-cloud-1729` \direct OIDC sign-in without a Hydra login_challenge is not yet supported\ (Cloudflare shows this as 502 from some POPs). The legacy Kratos selfservice OIDC flow initiated the Hydra login challenge internally; W2's `/oidc/start` had no equivalent, so the React `/login` page's \Sign in with...\ button forwarded to W2 without a `login_challenge` and OIDCCallback fell into its fail-closed branch.

### What changed

- `OIDCStart` now bounces server-side through Hydra's `/oauth2/auth` endpoint when the inbound URL lacks `?login_challenge=`, parses the freshly minted challenge out of the 302 `Location` header, and feeds it into the W2 state binding. The callback then has a valid challenge for `accept_login`, Hydra returns a code-carrying redirect to `/oauth/callback`, and `HandleRedirect` closes the loop via its existing code exchange + cookie issuance.
- The off-eTLD guard moved ahead of the new bounce so off-eTLD-custom-domain misconfigs surface the correct `meshery-cloud-1725` error instead of being masked by a bounce failure.
